## 💡 核心要點速覽

### 架構五層

```
層級 1：主機層       Windows Docker Desktop + K8s
層級 2：基礎層       Redis, MySQL, MinIO, Bridge
層級 3：應用層       Backend, Frontend, Worker
層級 4：通信層       Service + Port Forwarding
層級 5：用戶層       Web UI + API Clients
```

### 啟動順序

```
1️⃣ 驗證依賴 (Docker, K8s, ComfyUI)
2️⃣ 部署基礎 (Redis, MySQL, PVC)
3️⃣ 部署應用 (Backend, Frontend, Worker)
4️⃣ 設置轉發 (Port Forward)
5️⃣ 驗證就緒 (Health Check)
```

### 工作流處理流程

```
用戶提交
  ↓
Backend 接收 → 驗證 → 保存到 Redis/MySQL
  ↓
Worker 輪詢任務 (Redis FIFO)
  ↓
Worker 調用 ComfyUI API
  ↓
ComfyUI 執行工作流 (GPU)
  ↓
Worker 保存輸出 → 更新 MySQL
  ↓
Frontend 輪詢狀態 → 顯示進度
  ↓
用戶下載結果
```

## ❓ 常問問題

**Q: 去哪裡查看某個特定的 API？**  
A: K8s_Comprehensive_Testing_Guide.md 第 7 節，有完整 API 端點列表和測試命令。

**Q: 關系統時數據會丟失嗎？**  
A: 不會。所有數據都保存在 MySQL（PVC 持久化）和 outputs 文件夾。第 4 節有安全關閉程序。

**Q: 如何在本地調試 Backend？**  
A: 第 6 節有詳細的交互式調試說明，可進入 Pod 內進行 Python 調試。

**Q: 性能不夠快怎麼辦？**  
A: 第 9 節有完整的性能基準和優化指南。

**Q: 如何為生產環境部署？**  
A: 參考 HYBRID_DEPLOYMENT_STRATEGY.md 和其他部署文檔。
